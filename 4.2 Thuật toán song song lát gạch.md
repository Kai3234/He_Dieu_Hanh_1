# Ý tưởng cơ bản của Tiling&Blocking
- Chia bộ nhớ toàn cục thành các ô nhỏ (tiles).
    
- Các luồng tập trung tính toán trên một hoặc một số ô nhỏ tại một thời điểm.
    
- Sử dụng bộ nhớ trên chip để lưu trữ ô nhỏ thay vì truy cập trực tiếp bộ nhớ toàn cục.
- 
![[Pasted image 20250218145414.png]]
This slide illustrates the basic concept of tiled parallel algorithms. When multiple threads access the same global memory locations, we can reduce the number of accesses to the global memory by loading the global memory contents into the on-chip memory once, and then have all the threads to access their input operands from the on-chip memory.

Usually, the global memory data is too big to fit into the on-chip memory. So it's important that we divide up the global memory contents into piles. In some cases, in the literature, they are referred to as blocks. Unfortunately, blocks conflict with the name thread blocks. So we're going to be using the term piles and piling in this course.

We also need to divide up the execution of the threads into phases, so that each phase will focus on a particular tile of the memory content. So if we can have all the threads to execute the same phase, then we can just load the tile that will be consumed by all these threads in that particular phase into the on-chip memory.

and then all the threads will consume their input operand from the on-chip memory during that phase.

# Khái niệm cơ bản của Tiling
- Trong một hệ thống giao thông đông đúc, giảm số lượng xe cộ đáng kể có thể cải thiện độ trễ.
    
- Ví dụ:
    
    - Đi chung xe (Carpooling) giúp giảm số xe trên đường.
        
    - Tiling giúp giảm số lần truy cập bộ nhớ toàn cục.
        
- Liên hệ với lập trình:
    
    - Tài xế = Luồng truy cập bộ nhớ.
        
    - Xe = Yêu cầu truy cập bộ nhớ.

# Các bước áp dụng kỹ thuật Tiling
1. Xác định một ô trong bộ nhớ toàn cục mà nhiều luồng cần truy cập.
    
2. Nạp ô đó từ bộ nhớ toàn cục vào bộ nhớ trên chip.
    
3. Sử dụng đồng bộ hóa rào cản để đảm bảo tất cả các luồng sẵn sàng trước khi bắt đầu giai đoạn tính toán.
    
4. Các luồng truy cập dữ liệu từ bộ nhớ trên chip thay vì bộ nhớ toàn cục.
    
5. Sử dụng đồng bộ hóa rào cản để đảm bảo tất cả các luồng hoàn thành giai đoạn hiện tại.
    
6. Chuyển sang ô tiếp theo và lặp lại quá trình.
